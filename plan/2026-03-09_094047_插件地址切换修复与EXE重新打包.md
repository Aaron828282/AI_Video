# 插件地址切换修复与EXE重新打包

生成时间：2026-03-09 09:40:47

## 任务背景
用户希望插件在默认走线上地址的同时，能够在插件内切换到本地地址进行采集上传，并保持本地地址不被自动覆盖；同时要求重新打包 EXE 供本地站点运行。

## 方案详情

### 现状分析
- 当前插件默认地址为线上站点，但自动迁移逻辑会把本地 8790 也重置回线上，影响本地调试。
- 插件暂未提供明确的面板内地址切换操作入口。
- 项目支持 electron-builder，可产出 Windows EXE。

### 技术方案
1. 调整迁移规则：仅迁移错误的前端地址（5173），不迁移本地后端地址（8790）。
2. 在插件侧边栏新增“接收地址”设置区，提供线上/本地/自定义切换与保存。
3. 更新 README 使用说明，明确切换方式。
4. 重新打包插件 ZIP，确保下载包与最新逻辑一致。
5. 执行 EXE 重新打包，生成最新桌面端产物。

### 影响范围
- extension/sidepanel.js
- extension/sidepanel.html
- extension/sidepanel.css
- extension/manifest.json
- README.md
- downloads/ai-auto-1688-extension.zip
- release/（打包产物目录）

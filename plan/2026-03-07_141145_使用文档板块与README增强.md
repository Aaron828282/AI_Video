# 使用文档板块与 README 增强

生成时间：2026-03-07 14:11:45
文件名：2026-03-07_141145_使用文档板块与README增强.md

## 任务背景
当前网站左侧导航仅包含商品库、AI分析结果、API Key管理，缺少面向最终用户的站内使用说明入口。README 偏向项目说明，用户侧功能与操作流程描述不够集中。目标是在不改动页面风格的前提下，补齐“使用文档”入口与内容，降低使用门槛。

## 方案详情

### 现状分析
- 前端入口状态由 activeSection 管理，当前仅支持 library、aiResults、apiKeys。
- 左侧菜单渲染逻辑集中在 web/src/App.tsx，可直接新增导航项。
- 页面标题与副标题通过 pageHeader 统一生成，新增板块后需补充标题映射。
- 现有 README.md 包含功能概览与部署说明，但缺少站内操作路径与用户使用流程。

### 技术方案
1. 在 web/src/components 新增 UsageGuideSection.tsx，承载站内使用说明内容。
2. 扩展 App.tsx 的 activeSection 联合类型与侧边栏菜单，新增“使用文档”入口。
3. 在主内容区新增“使用文档”分支，复用当前页面框架，保持视觉风格一致。
4. 在 styles.css 新增文档板块样式（仅结构与可读性，不改全局主题）。
5. 更新 README.md，强化“网站功能说明 + 用户操作步骤 + 常见流程”文档内容。

技术取舍：采用静态组件而非 Markdown 运行时渲染，不增加额外依赖，降低实现复杂度和维护成本。

### 影响范围
- web/src/App.tsx
- web/src/styles.css
- web/src/components/UsageGuideSection.tsx（新增）
- README.md
- plan/2026-03-07_141145_使用文档板块与README增强.md（本方案文档）

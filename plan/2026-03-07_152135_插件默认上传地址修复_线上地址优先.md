## 任务背景
用户反馈新下载插件后，采集数据被上传到本地开发地址（http://localhost:5173），未进入 Railway 线上网站数据库。需要修复插件默认上传目标并兼容历史错误配置。

## 方案详情

### 现状分析
- 插件默认 API 基础地址仍是本地地址。
- 插件运行时优先读取 chrome.storage.sync 中的 apiBase，历史错误值会持续生效。
- 线上网站地址已固定为 https://ai-auto-1688-server-production.up.railway.app。

### 技术方案
1. 将插件默认 API 基础地址改为线上 Railway 域名。
2. 新增历史地址迁移逻辑：若检测到 localhost/127.0.0.1 的开发地址（尤其 5173），自动切换到线上地址并回写存储。
3. README 补充默认行为与手动切换到本地开发的方式。
4. 重新打包插件 ZIP，确保用户下载的是修复版本。
5. 提交并推送到 GitHub，触发 Railway 自动部署。

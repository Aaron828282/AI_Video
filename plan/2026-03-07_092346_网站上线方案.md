# 网站上线方案（内测到正式公网）

生成时间：2026-03-07 09:23:46
文件名：2026-03-07_092346_网站上线方案.md

## 任务背景
当前项目为 server + web + extension 的单仓结构，后端可托管前端静态资源，已具备本地运行能力，但尚未形成生产部署闭环。你希望将该项目上线为可访问网站，并明确当前尚未具备用户注册/登录功能，需要在上线策略中给出可落地的分阶段处理方案。

## 方案详情

### 现状分析
- 项目结构：Node.js/Express 后端 + React/Vite 前端 + 浏览器插件。
- 前后端关系：后端可直接托管 web/dist，可做单域名部署。
- 当前约束：前端与插件默认 API 地址为 http://localhost:8787，需生产化配置。
- 安全风险：/static 当前映射整个 DATA_DIR，存在敏感文件暴露风险。
- 鉴权现状：业务接口未实现用户级注册/登录鉴权，当前仅有 API Key 管理页密码会话。
- 数据层现状：以本地 JSON 文件持久化，适合单机部署，不适合多实例并发写。

### 技术方案
采用“两期上线策略”：

#### 一期（受控内测上线）
目标：在不立即建设完整用户系统的前提下，先实现可访问、可验证、可运维的线上版本。

实施要点：
1. 前端与插件 API 基址改为可配置（环境变量/构建注入），移除 localhost 硬编码。
2. 后端静态资源暴露最小化，仅公开图片等必要目录，阻断敏感配置文件访问。
3. 在网关层增加访问门禁（Basic Auth / IP 白名单 / Access 网关）与 HTTPS。
4. 使用单机部署（Docker Compose 或 PM2+Nginx），挂载持久化数据目录。
5. 增加发布、健康检查、日志与回滚脚本，形成可重复发布流程。

#### 二期（正式公网上线）
目标：补齐用户能力后放开公开访问。

实施要点：
1. 建设用户注册/登录（邮箱或手机号）与密码安全策略。
2. 引入 JWT/Session 鉴权中间件，对业务接口进行权限校验。
3. 数据模型按用户隔离（用户数据分区、审计日志、限流策略）。
4. 迁移持久化方案（建议数据库）以支持并发与扩展。
5. 灰度放量后移除一期外层门禁策略。

### 影响范围
- 后端：server/src/index.js（静态资源路由、安全中间件、接口访问控制）
- 前端：web/src/App.tsx（API_BASE 配置化）
- 插件：extension/sidepanel.js、extension/manifest.json（API 域名与 host 权限）
- 配置：server/.env、部署环境变量、反向代理配置
- 运维：部署脚本、日志、备份与回滚流程

## 原子步骤清单

### 步骤 1：确定上线边界与阶段目标
- **操作对象**：上线策略
- **具体动作**：明确一期“受控内测”、二期“正式公网”标准
- **预期结果**：上线目标清晰，避免无鉴权直接公网暴露
- **关键里程碑**：是

### 步骤 2：前端/插件 API 基址生产化
- **操作对象**：web/src/App.tsx、extension/sidepanel.js、extension/manifest.json
- **具体动作**：将 API 地址改为可配置项，支持生产域名
- **预期结果**：生产环境请求不再指向 localhost
- **关键里程碑**：否

### 步骤 3：收敛静态文件暴露范围
- **操作对象**：server/src/index.js
- **具体动作**：将 /static 从“整个 DATA_DIR”调整为“仅必要子目录”
- **预期结果**：敏感配置文件不可被公网读取
- **关键里程碑**：是

### 步骤 4：部署反向代理与 HTTPS
- **操作对象**：Nginx/Caddy
- **具体动作**：配置 TLS、反向代理、访问门禁、基础限流
- **预期结果**：外网访问安全可控
- **关键里程碑**：是

### 步骤 5：部署后端服务与持久化目录
- **操作对象**：服务器运行环境
- **具体动作**：部署 Node 服务，挂载持久化数据卷，启用进程守护
- **预期结果**：服务可持续运行并重启恢复
- **关键里程碑**：否

### 步骤 6：上线发布与回滚流程固化
- **操作对象**：发布脚本/CI
- **具体动作**：标准化构建、发布、健康检查、回滚
- **预期结果**：发布过程可重复、可审计
- **关键里程碑**：否

### 步骤 7：监控与告警接入
- **操作对象**：日志与监控系统
- **具体动作**：接入请求日志、错误率与响应时延告警
- **预期结果**：故障可发现、可定位
- **关键里程碑**：否

### 步骤 8：建设用户注册登录与鉴权
- **操作对象**：后端鉴权模块 + 前端登录流程
- **具体动作**：实现注册登录、令牌验证、接口权限控制
- **预期结果**：满足正式公网开放条件
- **关键里程碑**：是

### 步骤 9：用户数据隔离与存储升级
- **操作对象**：数据模型与存储层
- **具体动作**：引入数据库并建立用户隔离策略
- **预期结果**：支持并发访问与后续扩展
- **关键里程碑**：否

### 步骤 10：灰度放量并正式上线
- **操作对象**：网关策略与发布策略
- **具体动作**：逐步放量，验证稳定后移除内测门禁
- **预期结果**：安全平滑切换为公网正式运行
- **关键里程碑**：是

## 预期结果
- 短期（一期）：网站在受控条件下稳定可访问、具备可运维与可回滚能力。
- 中期（二期）：完成用户注册登录与接口鉴权，具备正式公网发布条件。
- 长期：通过存储升级与监控体系支撑业务增长与持续迭代。
